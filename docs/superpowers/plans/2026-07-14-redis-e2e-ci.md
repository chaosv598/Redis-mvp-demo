# Redis End-to-End CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a PR-blocking GitHub Actions path that fetches the Redis 7.0.15 pinned commit, applies a portable demo patch, compiles and starts Redis, runs functional checks, and enforces a small `redis-benchmark` health threshold.

**Architecture:** Keep `tools/verify.sh` as the repository's only developer command and add an `--e2e redis-7.0.15` mode. A test-only patch changes the runtime version marker, while the existing single GitHub Actions job runs both fast verification and E2E verification and publishes diagnostics.

**Tech Stack:** Bash, Git, Python 3 + PyYAML, GNU Make/GCC, Redis 7.0.15, GitHub Actions, `redis-cli`, `redis-benchmark`.

## Global Constraints

- Preserve the existing no-argument behavior of `bash tools/verify.sh`.
- Keep one version metadata file, one developer tool, and one GitHub Actions job.
- Read upstream repository and exact commit from `versions/redis-7.0.15/version.yaml`; never fall back to a tag in E2E mode.
- Correct the invalid mainline Redis 7.0.15 SHA to the official lightweight tag target `f35f36a265403c07b119830aa4bb3b7d71653ec9` before E2E execution.
- Keep the demo patch under `tests/fixtures/`; never add it to product `patches[]` metadata.
- Use GitHub-hosted `ubuntu-latest`; do not claim KRAIO/DTOE validation.
- Require PING_INLINE throughput of at least 10,000 requests/s; SET and GET are informational.
- Do not commit build products, logs, result directories, credentials, or downloaded Redis sources.

- [ ] **Step 4: Implement metadata and E2E lifecycle functions**

Add focused functions before ordinary verification begins:

```bash
read_e2e_metadata() {
    python3 - "$ROOT/versions/$1/version.yaml" <<'PY'
import sys, yaml
data = yaml.safe_load(open(sys.argv[1], encoding="utf-8"))
upstream = data["upstream_base"]
print(upstream["repo"])
print(upstream["commit"])
PY
}

wait_for_redis() {
    local cli=$1 port=$2
    for _ in $(seq 1 30); do
        "$cli" -p "$port" PING 2>/dev/null | grep -qx PONG && return 0
        sleep 0.2
    done
    return 1
}

run_e2e() {
    local version=$1
    local version_dir="$ROOT/versions/$version"
    local patch="$ROOT/tests/fixtures/$version/0001-ci-version-marker.patch"
    local work results source port pid=0
    work=$(mktemp -d)
    results=${E2E_RESULTS_DIR:-$work/results}
    source="$work/redis"
    port=${REDIS_E2E_PORT:-16379}
    mkdir -p "$results"
    cleanup() {
        if [ "$pid" -gt 0 ] && kill -0 "$pid" 2>/dev/null; then kill "$pid"; wait "$pid" 2>/dev/null || true; fi
        rm -rf "$work"
    }
    trap cleanup EXIT

    mapfile -t metadata < <(read_e2e_metadata "$version")
    local repo=${metadata[0]} sha=${metadata[1]}
    echo "E2E metadata: $version repo=$repo commit=$sha"
    [ "${VERIFY_E2E_DRY_RUN:-0}" = 1 ] && return 0

    git init -q "$source"
    git -C "$source" remote add origin "$repo"
    git -C "$source" fetch --quiet --depth 1 origin "$sha"
    git -C "$source" checkout --quiet --detach FETCH_HEAD
    [ "$(git -C "$source" rev-parse HEAD)" = "$sha" ]
    git -C "$source" apply --check "$patch"
    git -C "$source" apply "$patch"

    make -C "$source" -j2
    "$source/src/redis-server" --version | grep -Fq 'v=7.0.15-ci-patched'

    "$source/src/redis-server" --port "$port" --bind 127.0.0.1 --save '' \
        --appendonly no --protected-mode yes --logfile "$results/redis.log" &
    pid=$!
    wait_for_redis "$source/src/redis-cli" "$port" || { cat "$results/redis.log" >&2; return 1; }
    [ "$("$source/src/redis-cli" -p "$port" SET ci:key value)" = OK ]
    [ "$("$source/src/redis-cli" -p "$port" GET ci:key)" = value ]
    "$source/src/redis-cli" -p "$port" DEL ci:counter >/dev/null
    "$source/src/redis-cli" -p "$port" INCR ci:counter >/dev/null
    [ "$("$source/src/redis-cli" -p "$port" INCR ci:counter)" = 2 ]

    "$source/src/redis-benchmark" -h 127.0.0.1 -p "$port" --csv \
        -t ping_inline,set,get -n 20000 -c 20 >"$results/benchmark.csv"
    local ping_rps
    ping_rps=$(awk -F, '$1 ~ /PING_INLINE/ {gsub(/"/, "", $2); print $2; exit}' "$results/benchmark.csv")
    [ -n "$ping_rps" ]
    awk -v actual="$ping_rps" 'BEGIN { exit !(actual >= 10000) }'
    printf '# Redis E2E CI\n\n- Version: `%s`\n- Commit: `%s`\n- Patch marker: `7.0.15-ci-patched`\n- Functional tests: PASS\n- PING_INLINE: %s requests/s\n' \
        "$version" "$sha" "$ping_rps" >"$results/summary.md"
    if [ -n "${GITHUB_STEP_SUMMARY:-}" ]; then cat "$results/summary.md" >>"$GITHUB_STEP_SUMMARY"; fi
}
```
